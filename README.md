# پلتفرم سلامت — نسخه بازتعریف‌شده

وب‌سایت راهبرد و طرح محصول برای یک خانواده سرویس‌های مستقل سلامت مبتنی بر هوش مصنوعی.

تز اصلی این نسخه:

> محصول عمودی اول؛ پلتفرم بعداً.

هر سرویس روی یک نیاز مشخص تمرکز می‌کند، جداگانه ساخته و اعتبارسنجی می‌شود و فقط پس از بلوغ، با رضایت روشن کاربر می‌تواند به سرویس‌های دیگر متصل شود. این پروژه در وضعیت فعلی یک پروپوزال مفهومی است؛ بک‌اند، احراز هویت، پایگاه داده یا خدمت پزشکی عملیاتی ندارد.

## ساختار سایت

- `/` — روایت اصلی و مدل برند مادر
- `/model` — مرز میان برند، سرویس عمودی و هسته مشترک آینده
- `/services` — خانواده سرویس‌ها، وضعیت و مرز هر مسیر
- `/roadmap` — نقشه راه شرط‌محور از پژوهش اولیه تا اتصال
- `/trust` — نقش AI، نظارت انسانی، ایمنی و رضایت داده
- `/blueprint` — معیار انتخاب سرویس اول، پایلوت پیشنهادی و تصمیم‌های باز

Routing با `HashRouter` انجام می‌شود تا روی GitHub Pages بدون تنظیم server fallback کار کند.

## تصمیم‌های محتوایی اصلی

- نام «پلتفرم سلامت» نام کاری است؛ برند نهایی هنوز انتخاب نشده.
- اولین سرویس قطعی نشده؛ «تغذیه و عادت‌های روزمره برای بزرگسالان کم‌خطر» فقط نمونه‌ای برای نمایش روش دامنه‌بندی است.
- همه سرویس‌های دیگر به‌عنوان مسیر آینده معرفی می‌شوند، نه قابلیت فعال.
- AI نقش اطلاعاتی و تصمیم‌یار دارد؛ تشخیص، تجویز و درمان مستقل خارج از دامنه‌اند.
- اشتراک داده میان سرویس‌ها پیش‌فرض نیست و در آینده نیز به رضایت صریح، محدود و قابل لغو نیاز دارد.
- تأمین دارو، در صورت ساخت، فقط از مسیر نسخه معتبر و داروخانه دارای مجوز و مطابق مقررات روز تعریف می‌شود.

منبع حقیقت محصول در [docs/PRODUCT_BLUEPRINT.md](./docs/PRODUCT_BLUEPRINT.md) و تصمیم‌های مهاجرت محتوا در [docs/CONTENT_MIGRATION.md](./docs/CONTENT_MIGRATION.md) ثبت شده‌اند.

## اجرا و بررسی

نیازمندی: Node.js `20.19+`؛ نسخه پیشنهادی Node 22.

```bash
npm ci
npm run dev
```

بررسی کامل:

```bash
npm run check
npm audit --audit-level=high
```

Build تولید:

```bash
npm run build
npm run preview
```

## استقرار

`vite.config.js` مقدار base را روی `/Health_Platform/` نگه می‌دارد. Push به `main` پس از lint، test، build و audit خروجی را در شاخه `gh-pages` منتشر می‌کند.

نشانی عمومی: <https://online6731.github.io/Health_Platform/>

## امنیت و حریم خصوصی

- هیچ کلید API یا متغیر محرمانه‌ای نباید با پیشوند `VITE_*` وارد فرانت‌اند شود.
- GitHub Pages فقط برای پروپوزال عمومی مناسب است، نه پردازش داده سلامت.
- ساخت خدمت واقعی به backend امن، کنترل دسترسی، مدیریت رضایت، audit log، سیاست نگه‌داری داده و ارزیابی ایمنی مستقل نیاز دارد.
- گزارش آسیب‌پذیری را طبق [SECURITY.md](./SECURITY.md) انجام دهید.

## وضعیت حقوقی و پزشکی

این مخزن ابزار پزشکی، داروخانه، نسخه‌نویس یا سرویس اورژانسی نیست. متن‌های ایمنی جایگزین مجوز، پروتکل بالینی، ارزیابی محصول یا مشاوره حقوقی حوزه قضایی هدف نمی‌شوند.
