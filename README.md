# پلتفرم سلامت — پروپوزال سرمایه‌گذاری Pre-seed

پرونده تعاملی سرمایه‌گذاری برای ساخت مرحله‌ای یک خانواده از سرویس‌های مستقل سلامت مبتنی بر هوش مصنوعی.

تز اصلی:

> اول یک کسب‌وکار سلامت قابل‌دفاع؛ بعد یک پلتفرم.

این دور برای ساخت یک سوپراپ بزرگ طراحی نشده است. سرمایه در هر مرحله برای حذف یک ریسک مشخص آزاد می‌شود: مسئله، تقاضای پولی، نتیجه کاربر، ایمنی، عملیات، اقتصاد و سپس تکرارپذیری. سرویس دوم و هسته مشترک تنها پس از عبور مستقل محصول اول از گیت‌های از پیش تعریف‌شده آغاز می‌شوند.

## وضعیت واقعی

- مرحله: تعریف و اعتبارسنجی پیش از MVP
- روایت سرمایه‌گذاری: Pre-seed کشف و اثبات
- سرویس اول: هنوز قطعی نشده؛ تغذیه کم‌خطر فقط Candidate پژوهش است
- بازار نخست: نیازمند تصمیم بنیان‌گذار
- محصول، کاربر، درآمد و traction: در این مخزن ادعا نشده‌اند
- بودجه ۸۵ میلیارد تومان: خروجی سناریوی برنامه‌ریزی ۲۴ماهه با چهار ماه پیش‌درآمد، نه درخواست نهایی یا عملکرد واقعی

همه اعداد و گزاره‌های مهم با یکی از برچسب‌های واقعیت فعلی، فرضیه، هدف، سناریو یا تصمیم باز ارائه می‌شوند.

## ساختار سایت

- `/` — خلاصه مدیریتی و وضعیت دور
- `/investor` — یادداشت کامل سرمایه‌گذاری: مسئله، شواهد، بازار، رقابت، درآمد، GTM، moat، تیم، ریسک، سرمایه و منطق بازده
- `/blueprint` — scorecard انتخاب Wedge، دامنه Candidate و تصمیم‌های باز
- `/roadmap` — برنامه دقیق ۲۴ماهه، ده هفته کشف/Concierge، گیت‌ها، outcome، KPI و حاکمیت
- `/financials` — ماشین‌حساب اقتصاد واحد مشتق از مدل رشد، SOM ظرفیت‌محور، سه سناریوی ۲۴ماهه، حساسیت و سرمایه
- `/trust` — زنجیره انتشار AI، عملیات ایمنی، داده، رضایت و مسیر حقوقی
- `/dataroom` — نقشه Due Diligence، سطح دسترسی، سیاست ادعا و اسناد لازم
- `/print` — پرونده یکپارچه برای چاپ یا ذخیره PDF
- `/model` — معماری عمودی اول و هسته مشترک آینده
- `/services` — سبد مفهومی سرویس‌های مستقل

Routing با `HashRouter` انجام می‌شود تا روی GitHub Pages بدون server fallback کار کند.

## اجرای محلی و کنترل کیفیت

نیازمندی: Node.js `20.19+`؛ نسخه پیشنهادی Node 22.

```bash
npm ci
npm run dev
```

بررسی کامل:

```bash
npm run check
npm audit --audit-level=high
```

Build تولید:

```bash
npm run build
npm run preview
```

## امنیت، پزشکی و اطلاعات محرمانه

- GitHub Pages فقط برای پروپوزال عمومی است و برای داده سلامت، cap table، قرارداد محرمانه یا اطلاعات هویتی استفاده نمی‌شود.
- هیچ کلید API، token یا credential نباید در فرانت‌اند یا تاریخچه عمومی قرار گیرد.
- این مخزن ابزار پزشکی، نسخه‌نویس، داروخانه یا سرویس اورژانسی نیست.
- نقش AI محدود به جریان‌های تعریف‌شده و تصمیم‌یاری است؛ تشخیص، تجویز و درمان مستقل خارج از دامنه‌اند.
- قوانین و مجوزهای بازار هدف باید با مشاور واجد صلاحیت و یادداشت حقوقی مکتوب بررسی شوند.

منابع WHO، NIST و FDA در سایت نقش چارچوب مرجع دارند و نشان تأیید، مجوز یا انطباق محسوب نمی‌شوند.
