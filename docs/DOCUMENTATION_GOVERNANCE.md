# حاکمیت محتوای پلتفرم سلامت

تاریخ بازتعریف: تیر ۱۴۰۵ (ژوئیه ۲۰۲۶)
وضعیت: `Concept — not an operational medical service`

## منبع حقیقت

ترتیب اعتبار اسناد این مخزن:

1. `docs/PRODUCT_BLUEPRINT.md`
2. محتوای منتشرشده در `src/content/siteContent.js`
3. `docs/CONTENT_MIGRATION.md`
4. README و صفحات رابط

خروجی‌های خام گفتگو، فصل‌های ۷۸گانه و معماری قدیمی از نسخه فعال حذف شده‌اند و منبع تصمیم محصول نیستند.

## وضعیت‌های مجاز ادعا

- `Concept`: جهت یا فرضیه پیشنهادی؛ هنوز پیاده‌سازی نشده.
- `Candidate`: گزینه پیشنهادی برای پژوهش اولیه یا پایلوت؛ تصمیم نهایی نیست.
- `Planned`: مالک، دامنه، معیار پذیرش و بودجه تأییدشده دارد.
- `Implemented`: به نسخه کد و محیط قابل بازتولید متصل است.
- `Validated`: روش، داده، معیار، نتیجه و محدودیت بازبینی شده‌اند.
- `Operational`: خدمت واقعی، مسئولیت، پشتیبانی، مجوزهای لازم و پایش رخداد دارد.

در نسخه فعلی فقط `Concept` و `Candidate` قابل استفاده‌اند.

## قواعد انتشار

- «فعال»، «ایمن»، «دقیق»، «مؤثر»، «مطابق» و درصدهای عملکرد بدون مدرک منتشر نمی‌شوند.
- عنوان‌هایی مانند «پزشک هوشمند» یا «روان‌شناس هوشمند» استفاده نمی‌شوند.
- «تحت نظارت متخصص» فقط وقتی مجاز است که متخصص واقعاً همان خروجی را پیش از مصرف بازبینی کند.
- سوپراپ، پرونده یکپارچه و هسته مشترک به‌عنوان افق احتمالی معرفی می‌شوند، نه محصول امروز.
- اتصال داده باید هدف، حداقل داده، دریافت‌کننده، مدت و راه لغو را روشن کند.
- دارو، نسخه و خدمت تخصصی فقط با نقش روشن فرد یا مرکز دارای صلاحیت توصیف می‌شوند.

## مسیر تصمیم

`فرضیه → تحقیق → دامنه مجاز → نیازمندی → پایلوت → شاهد → شرط ادامه/توقف`

هر حلقه مفقود، وضعیت ادعا را در `Concept` نگه می‌دارد.
